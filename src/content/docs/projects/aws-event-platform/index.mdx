## Overview

A cloud-native asynchronous processing platform built on AWS.

Incoming client requests are validated by an API Gateway + Lambda ingestion layer. Jobs are enqueued into an SQS queue with a dead-letter queue (DLQ) configured. Containerized Python workers deployed on AWS ECS Fargate poll the queue, process jobs, and emit structured logs to CloudWatch.

This architecture emphasizes reliability with:
- at-least-once delivery semantics,
- visibility timeout-safe processing,
- poison message quarantine in a DLQ,
- and clear observability into success or failure paths.

## Tech Stack
- AWS: API Gateway, Lambda, SQS, DLQ, ECS Fargate, ECR, CloudWatch
- Python, boto3, Docker
- CI/CD (image build + deployment pipeline)

## Highlights
- Designed and deployed resilient AWS infrastructure for job queuing and background workers
- Workers implement idempotency and safe delete-after-success
- Operational logs and metrics centralized in CloudWatch for troubleshooting
- Demonstrates infra-level responsibility expected in real DevOps workflows
