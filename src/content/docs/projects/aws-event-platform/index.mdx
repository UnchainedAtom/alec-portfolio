## Overview

An event-driven asynchronous processing system built on AWS to explore reliability patterns used in production microservice architectures.

Requests are accepted via API Gateway and validated in a Lambda ingestion layer. Jobs are enqueued to SQS with a dead-letter queue configured for failure isolation. Containerized Python workers deployed on ECS Fargate poll the queue, process jobs, and emit logs to CloudWatch.

## Architecture
![AWS Architecture Diagram](/images/awsEventPlatform/aws-architecture.png)

Client  
→ API Gateway  
→ Lambda (validation + enqueue)  
→ SQS (with DLQ)  
→ ECS Fargate worker  
→ CloudWatch Logs  

## Deployment (ECS Fargate)

![ECS Running Service](/images/awsEventPlatform/ecs-running.png)

- Docker image built locally
- Image pushed to Amazon ECR
- ECS Fargate service deployed with task role separation
- Environment variables injected securely
- Long polling enabled for worker efficiency

## Observability & Runtime Behavior

![CloudWatch Logs](/images/awsEventPlatform/cloudwatch-logs.png)

Workers implement:

- Visibility-timeout-safe processing
- Delete-after-success pattern
- At-least-once delivery semantics
- Idempotency key generation
- Dead-letter queue isolation for poison messages

Logs are streamed to CloudWatch for operational debugging.

## Operational Considerations

- Queue visibility timeout configured to prevent duplicate processing
- DLQ used to isolate failed messages after max receive attempts
- IAM roles separated for Lambda and ECS tasks
- Containerized worker supports horizontal scaling via ECS service scaling
