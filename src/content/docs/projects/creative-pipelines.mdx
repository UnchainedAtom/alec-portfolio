# Creative Pipelines & Systems Projects

DevOps + infrastructure thinking extends beyond production services. Creative pipelines, build systems, and hardware automation present the same reliability, observability, and automation challenges as any distributed system.

## Unreal Engine Pipeline & Asset Workflows

### Problem
AAA game development requires standardized asset pipelines: MetaHuman customization, material layering, kitbashing workflows, and naming conventions must stay consistent across large teams. Manual processes lead to naming conflicts, broken dependencies, and iteration friction.

### Solution
- Structured MetaHuman workflows with consistent export targets
- Substance material layering with versioning and reusability
- Modular asset kits with clear dependency trees
- Validation scripts for asset completeness and naming standards

### Tools & Learning
- Unreal Engine 5, MetaHuman System, Substance 3D Painter
- Blueprint scripting for asset validation
- Asset naming conventions and metadata frameworks
- Iterative refinement under real production constraints

## 3D Asset Pipeline (Blender → Substance → UE)

### Problem
Artists moving assets through multiple tools lose context: which Blender file produced which exported mesh? Are high-poly sculpts cached or recomputed each time? How do you prevent incompatible material layer versions?

### Solution
- Organized Blender project structure with clear versioning
- ZBrush high-poly sculpts managed as source-of-truth assets
- Substance material definitions version-controlled and reusable
- Export validation (polycount, LOD chains, material slots)

### Tools & Learning
- Blender (modeling, LOD generation, rigging prep)
- ZBrush (high-poly sculpting, decimation workflows)
- Substance 3D Painter (material authoring, smart maps, export templates)
- UE material systems, LOD concepts, and asset optimization

## Home Automation & Orchestration

### Problem
Home devices (lights, sensors, controllers) are unreliable, have different APIs, and lack cohesive control. Manual integrations fail silently.

### Solution
- Home Assistant as central orchestration hub
- Automation rules for device health checks and failover
- Logging and monitoring for device state changes
- Visualization dashboard for system observability

### Tools & Learning
- Raspberry Pi (Linux, systemd, networking)
- Home Assistant (YAML automation, service orchestration)
- Local mqtt/REST API integration
- Debugging unstable IoT devices

## LED / Light Control System (Hardware + Software)

### Problem
Custom LED installations and dynamic lighting require reliable hardware control, synchronized state, and recovery from disconnections.

### Solution
- Microcontroller firmware for LED sequences
- Service layer abstracting hardware details
- State synchronization across distributed lights
- Web interface + monitoring for system health

### Tools & Learning
- Microcontroller programming (timing, state management)
- Network protocols (MQTT, REST, serial communication)
- Color spaces and rendering math
- Resilience patterns (reconnection logic, fallback states)

## Why this belongs in a DevOps portfolio
Creative pipelines are **distributed systems** with **failure modes**. They require:
- **Standardization** (consistent exports, naming, versioning)
- **Observability** (knowing when things break, tracking asset flow)
- **Automation** (validation, batch processing, dependency resolution)
- **Reliability** (tolerance for tool crashes, version mismatches, network hiccups)

These projects show that I can apply **operations mindset** beyond traditional cloud infrastructure—understanding systems deeply, building for failures, and automating repetitive safety checks.
